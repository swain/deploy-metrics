# CLAUDE.md — deploy-metrics

MTTD (Mean Time to Deploy) dashboard for GoodParty.org, hosted on GitHub Pages at https://swain.github.io/deploy-metrics/.

## What This Does

`update.ts` fetches PR data from 4 GoodParty repos (gp-webapp, gp-api, people-api, election-api) via the GitHub API, computes how long PRs take to go from `develop` to `master`, and publishes `metrics.json` + `index.html` to this repo.

## Running Locally

```bash
GH_TOKEN=<your-pat> npx tsx update.ts
```

Requires a PAT with `repo` scope. The script publishes directly to `swain/deploy-metrics` via the GitHub Contents API — there is no local output mode.

## How It Runs in CI

A GitHub Action (`.github/workflows/update.yml`) runs daily at 6pm ET. It uses the `GH_PAT` secret.

## Architecture

- **`update.ts`** — single self-contained script. Fetches data, computes metrics, publishes results.
- **`index.html`** — static dashboard (generated by the script). Loads `metrics.json` client-side, renders with Chart.js.
- **`metrics.json`** — computed MTTD data (generated by the script). Not checked in manually.

## How MTTD is Computed

1. Fetch PRs merged to `develop` (feature PRs) across all 4 repos
2. Fetch PRs merged to `master` (deploy events) — these are the develop→qa→master promotions
3. For each develop PR, check if its merge commit SHA exists in master's history (deployed)
4. If deployed, find the first PR-to-master merged after the develop PR — that's the deploy timestamp
5. MTTD = deploy timestamp - develop merge timestamp

## Key Details

- The dashboard HTML is embedded as a template string at the bottom of `update.ts`
- Weekly averages are grouped by ISO week of the develop merge date
- The chart uses a 4-week rolling average to smooth out weekend/release cadence noise
- `index.html` and `metrics.json` are committed via the GitHub Contents API, not git push
